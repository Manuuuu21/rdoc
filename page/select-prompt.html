      select(), prompt()

      The select() function is used to create a selectable menu for the invoking character, similar to the menu command, but it directly returns the selected option number.

      üìö Function Overview

      The select() function is a convenient alternative to the menu command for simpler cases where you need to present options and get a direct numerical response, such as a yes/no question. It displays a dialog box with clickable options.

      Here is the standard syntax:
      select("<option>"{,"<option>",...})

      Each <option> is a string that will be displayed as a clickable choice in the menu. Options can be grouped using a colon (:) as a separator, similar to the menu command.

      Example Code:
      //===== rAthena Script =======================================
//= Simple Selector
//===== By: rAthena AI Assistant ============================
//= Demonstrates the basic usage of the select() function.
//============================================================
payon,148,232,4	script	Simple Selector	567,{
    mes "[Simple Selector]";
    mes "Hello there, adventurer!";
    mes "Do you wish to proceed on your journey?";
    next;

    // Use select() to present options and get the player's choice.
    // Option 1: "Yes", Option 2: "No" the "else" statement
    .@choice = select("Yes:No");

    if (.@choice == 1) {
        mes "[Simple Selector]";
        mes "Excellent! May your path be filled with glory!";
    } else {
        mes "[Simple Selector]";
        mes "Understood. Take your time, the world awaits.";
    }
    close;
}

      In this script:

      This script defines a 'Simple Selector' NPC located in Payon. Its primary function is to greet the player, ask a yes/no question using the select() command, and then display message based on the player's choice.

      What is prompt()?

      The prompt() function in rAthena scripting is a convenient alternative to the menu() command, primarily used for simpler selection scenarios where a complex label structure is not required. It allows you to present selectable options to the player and captures their choice. 'prompt' works almost the same as select, except that when a character clicks the Cancel button, this function will return 255 instead.

      ü§î Purpose of prompt()

      • Display a selectable menu to the invoking character.
      • Return the numerical index of the option chosen by the player.
      • Set the special temporary character variable @menu to the selected option's index.
      • Distinguish itself from select() by returning 255 if the player clicks the 'Cancel' button, whereas select() would simply end the script or return 0 depending on context.

      üìú Syntax of prompt()

      The syntax for prompt() is as follows:

      prompt("<option>"{,"<option>",...})

      ‚öôÔ∏è Syntax Parameter Explanation

      "<option>": This is a string representing a selectable choice in the menu. You can provide multiple options, separated by commas.

      • Options can be grouped using the colon (:) character. For example, "Yes:No" will display 'Yes' and 'No' as separate options, but if 'Yes' is selected, @menu will be 1, and if 'No' is selected, @menu will be 2.
      • Empty strings as options will not be displayed, allowing for dynamic menu creation.
      Example Code:
      //===== rAthena Script =======================================
//= Prompt_Example
//===== By: rAthena AI Assistant ============================
//= Demonstrates the usage of the prompt() function.
//============================================================
payon,156,163,4	script	Prompt_NPC	567,{
    mes "Hello there! Do you enjoy rAthena scripting?";
    .@choice = prompt("Yes:No");

    if (.@choice == 1) {
        mes "That's wonderful! I'm glad you enjoy it.";
    } else if (.@choice == 2) {
        mes "Oh, I see. Perhaps I can help make it more enjoyable for you!";
    } else if (.@choice == 255) {
        mes "You cancelled the conversation. Come back if you change your mind.";
    }
    close;
}
