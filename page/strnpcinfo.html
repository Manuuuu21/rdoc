      strnpcinfo()

      ü§î What is strnpcinfo?

      strnpcinfo is a function in rAthena scripting that allows you to retrieve various parts of the name of the NPC object from which the script is currently running.

      üéØ Purpose of strnpcinfo?

      The primary purpose of strnpcinfo is to obtain identification and location details about the NPC itself. This is particularly useful for dynamic scripting where an NPC might need to refer to its own name (display or unique) or its map location, especially when dealing with multiple instances of the same NPC or when creating event labels dynamically.

      üìù Syntax of strnpcinfo?

      The syntax for strnpcinfo is as follows: strnpcinfo(<type>)

      üìã Syntax Parameter Explanation:

      The <type> parameter determines what specific information about the NPC will be returned. It can be one of the following integer values:

      • 0: Returns the NPC's full display name, which includes both the visible and hidden parts (e.g., "Display name#Unique name").
      • 1: Returns only the visible part of the NPC's display name.
      • 2: Returns the hidden part of the NPC's display name, often used for unique identification purposes.
      • 3: Returns the NPC's unique name (e.g., "::name"), which is used for internal identification.
      • 4: Returns the name of the map the NPC is currently located on.

      üí° How to use strnpcinfo?

      You can use strnpcinfo within any NPC script to get information about that specific NPC. It's commonly used in conjunction with mes to display information, or with set to store the information in a variable for later use.

      Example Code:
      //===== rAthena Script =======================================
//= NPC Info Display
//===== By: rAthena AI Assistant ============================
//= Demonstrates the usage of strnpcinfo to display NPC details.
//============================================================
splendide,194,145,5	script	NPC_Info_Bot#MyUniqueNPC	904,{
    mes "^FF0000[NPC Information Bot]^000000";
    mes "Hello, I can tell you about myself!";
    next;

    mes "My full display name is: ^0000FF" + strnpcinfo(0) + "^000000";
    mes "My visible name is: ^0000FF" + strnpcinfo(1) + "^000000";
    mes "My hidden name is: ^0000FF" + strnpcinfo(2) + "^000000";
    mes "My unique identifier is: ^0000FF" + strnpcinfo(3) + "^000000";
    mes "I am currently located on map: ^0000FF" + strnpcinfo(4) + "^000000";
    next;

    mes "I hope this information was helpful!";
    close;
}

      In this script:

      • The script defines an NPC named NPC_Info_Bot with a unique name UniqueBot located at prontera,150,150. The sprite ID is 767.
      • When a player interacts with this NPC, a dialogue box appears.
      • The script then uses strnpcinfo with different <type> values (0, 1, 2, 3, and 4) to retrieve and display:
        • The NPC's full display name (NPC_Info_Bot#UniqueBot).
        • The visible part of its name (NPC_Info_Bot).
        • The hidden part of its name (UniqueBot).
        • Its unique identifier (::NPC_Info_Bot#UniqueBot).
      • Each piece of information is displayed on a new line, demonstrating how strnpcinfo dynamically fetches these details from the NPC's own properties.
