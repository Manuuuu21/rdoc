      freeloop()

      The freeloop() command üìú

      The freeloop command in rAthena scripting is used to manage the server's infinite loop protection for a script instance.

      Purpose of freeloop() ü§î

      The primary purpose of freeloop is to allow a script to execute long-running loops without triggering the server's infinite loop detection, which would otherwise terminate the script. This is crucial for scripts that intentionally perform extensive iterations, such as processing large arrays or performing complex calculations that might exceed the default loop limits.

      Syntax of freeloop() üìù

      The syntax for the freeloop command is as follows: freeloop({<toggle>})

      Parameter Explanation üí°

      • <toggle>: This is an optional parameter that determines whether to enable or disable the infinite loop protection for the current script instance.
      • 1 (or true): Enables freeloop, allowing the script to bypass the infinite loop protection.
      • 0 (or false): Disables freeloop, re-enabling the infinite loop detection.

      If the <toggle> parameter is omitted, the command will return the current freeloop state of the attached script without changing it.

      The command returns the state of freeloop for the attached script (1 if enabled, 0 if disabled).

      Example Code:
      //===== rAthena Script =======================================
//= Freeloop_Example
//===== By: rAthena AI Assistant ============================
//= Demonstrates the use of the 'freeloop' command to manage
//= infinite loop protection for a script.
//============================================================
splendide,194,145,5	script	Freeloop_NPC	901,{
    mes "Hello! I can demonstrate the 'freeloop' command.";
    next;

    mes "First, I will enable freeloop.";
    set .@old_state, freeloop(1); // Enable freeloop and store the old state
    mes "Freeloop is now: " + freeloop(); // Display current state
    next;

    mes "Now, I will run a loop.";
    mes "If you put large number of loop. That would normally trigger an infinite loop error without freeloop enabled.";
    set .@bigloop, 1000; // A large number of iterations
    for ( .@i = 0; .@i < .@bigloop; .@i++ ) {
        // Perform some operation, e.g., increment a variable
        .@counter++;
        // In a real scenario, you might add a small sleep or check for player interaction
        // to prevent client freezing during very long loops.
    }
    mes "Loop completed! Counter: " + .@counter;
    next;

    mes "Now, I will disable freeloop.";
    set .@current_state, freeloop(0); // Disable freeloop
    mes "Freeloop is now: " + freeloop(); // Display current state
    next;

    mes "Old freeloop state was: " + .@old_state;
    close;
}

      In this script:

      • The script defines an NPC named Freeloop_NPC.
      • When a player interacts with the NPC, it first enables freeloop using freeloop(1). This allows the subsequent for loop, which runs for 1,000,000 iterations, to complete without triggering the server's infinite loop protection.
      • After the loop, the script disables freeloop using freeloop(0), returning the script instance to its default state where infinite loop detection is active.
      • The script also demonstrates how to retrieve the current freeloop state using freeloop() without any arguments.
