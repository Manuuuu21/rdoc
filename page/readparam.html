      readparam()

      What is readparam? ü§î

      The readparam function in rAthena scripting is used to retrieve specific statistical information about a character. It can return various stats for the invoking character or a specified player.

      How to use readparam? üìù

      You use readparam by calling it with a parameter number (or a constant representing a stat) to specify which piece of information you want to retrieve. Optionally, you can specify a character's name or ID to get information about another player.

      Purpose of readparam? üéØ

      The primary purpose of readparam is to allow scripts to access and utilize character statistics for various in-game functionalities. This includes checking player levels, HP/SP, zeny, and other attributes to create dynamic interactions, quest requirements, or conditional dialogues. While many of these stats are also available as direct variables (e.g., Zeny, BaseLevel), readparam offers a more generalized way to access them, especially when querying other characters.

      Syntax of readparam? üìú

      • readparam(<parameter number>{,"<character name>"})
      • readparam(<parameter number>{,<char_id>})

      Syntax Parameter Explanation üß©

      • <parameter number>: This is a numerical ID or a predefined constant representing the specific stat or parameter you want to retrieve. The rAthena documentation lists many common constants such as StatusPoint, BaseLevel, SkillPoint, Class, Upper, Zeny, Sex, Weight, MaxWeight, JobLevel, BaseExp, JobExp, NextBaseExp, NextJobExp, Hp, MaxHp, Sp, MaxSp, BaseJob, Karma, Manner, bVit, bDex, bAgi, bStr, bInt, bLuk, Ap, MaxAp.
      • "<character name>" (Optional): A string representing the name of the character whose information you want to retrieve. If omitted, the function will return information about the character currently attached to the script.
      • <char_id> (Optional): An integer representing the character ID of the player whose information you want to retrieve. This serves the same purpose as "<character name>" but uses the character's unique ID.
      Example Code:
      //===== rAthena Script =======================================
//= StatChecker
//===== By: rAthena AI Assistant ============================
//= Demonstrates the usage of the 'readparam' function to retrieve
//= and display various character statistics.
//============================================================
splendide,194,145,5	script	StatChecker	905,{
    mes "^FF0000[Stat Checker NPC]^000000";
    mes "Hello, " + strcharinfo(0) + "!";
    mes "I can tell you some interesting facts about yourself.";
    next;

    mes "^FF0000[Your Basic Stats]^000000";
    mes "Base Level: " + readparam(BaseLevel);
    mes "Job Level: " + readparam(JobLevel);
    mes "Current HP: " + readparam(Hp) + " / " + readparam(MaxHp);
    mes "Current SP: " + readparam(Sp) + " / " + readparam(MaxSp);
    mes "Zeny: " + readparam(Zeny);
    mes "Weight: " + readparam(Weight) + " / " + readparam(MaxWeight);
    next;

    mes "^FF0000[Your Attributes]^000000";
    mes "STR: " + readparam(bStr);
    mes "AGI: " + readparam(bAgi);
    mes "VIT: " + readparam(bVit);
    mes "INT: " + readparam(bInt);
    mes "DEX: " + readparam(bDex);
    mes "LUK: " + readparam(bLuk);
    next;

    mes "^FF0000[Other Information]^000000";
    mes "Your Class ID: " + readparam(Class);
    mes "Your Gender (0=Female, 1=Male): " + readparam(Sex);
    mes "Your Upper Class Type (0=Normal, 1=Advanced, 2=Baby): " + readparam(Upper);
    next;

    mes "I hope this information was helpful!";
    close;
}

      In this script:

      This script defines an NPC named 'StatChecker' located in Prontera at coordinates 150,150. When a player interacts with this NPC, it initiates a dialogue that provides various statistics about the player. The script uses the readparam function extensively to fetch these details:

      • It first greets the player by name using strcharinfo(0).
      • It then displays basic character information such as BaseLevel, JobLevel, current and maximum Hp and Sp, Zeny, and Weight using readparam with the corresponding constants.
      • Next, it retrieves and shows the player's core attributes: bStr, bAgi, bVit, bInt, bDex, and bLuk.
      • Finally, it provides additional details like the player's Class ID, Sex, and Upper class type.
      • The next; command is used to paginate the dialogue, allowing the player to read through the information in sections.
      • The close; command ends the NPC dialogue.

      This script effectively demonstrates how readparam can be used to access a wide range of character data directly within a script, enabling personalized and informative NPC interactions.
