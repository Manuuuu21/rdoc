      strcharinfo()

      ü§î What is strcharinfo?

      strcharinfo is a function in rAthena scripting that retrieves various pieces of information about the invoking character, such as their name, party name, or guild name.

      üí° How to use strcharinfo?

      This function is used by calling it with a specific type parameter, which determines what information it will return. It can be used within any script where a character's RID (account ID) is attached.

      üéØ Purpose of strcharinfo?

      Its primary purpose is to dynamically retrieve character-related string information within a script. This is useful for personalizing NPC dialogues, logging player actions, or creating conditional script flows based on a player's group affiliations.

      üìù Syntax of strcharinfo?

      The syntax for strcharinfo is as follows: strcharinfo(<type>{,<char_id>})

      ‚öôÔ∏è Syntax Parameter Explanation:

      • type: This is a numerical value that specifies the kind of information to be returned.
        • 0: Returns the character's name (string).
        • 1: Returns the name of the party the character is in (string). Returns an empty string if not in a party.
        • 2: Returns the name of the guild the character is in (string). Returns an empty string if not in a guild.
        • 3: Returns the name of the map the character is currently on (string).
      • char_id (optional): If provided, the function will return information for the specified character ID instead of the invoking character. If the character is not found or offline, it returns an empty string for name types and 0 for ID types.
      Example Code:
      //===== rAthena Script =======================================
//= InfoNPC
//===== By: rAthena AI Assistant ============================
//= Provides information about the player and other online players.
//============================================================
splendide,194,145,5	script	InfoNPC	902,{
    mes "Hello, adventurer!";
    mes "";
    mes "Your name is: " + strcharinfo(0) + ".";
    mes "You are currently on map: " + strcharinfo(3) + ".";

    if (strcharinfo(1) != "") {
        mes "You are in party: " + strcharinfo(1) + ".";
    } else {
        mes "You are not currently in a party.";
    }

    if (strcharinfo(2) != "") {
        mes "You are a member of guild: " + strcharinfo(2) + ".";
    } else {
        mes "You are not currently in a guild.";
    }

    next;
    mes "I can also tell you about other players if you know their name.";
    mes "What is the name of the player you want to check?";
    input .@target_name$;

    .@target_char_id = getcharid(0, .@target_name$);

    if (.@target_char_id == 0) {
        mes "Sorry, I cannot find a player named " + .@target_name$ + " online.";
    } else {
        mes "Information for " + .@target_name$ + ":";
        mes "Map: " + strcharinfo(3, .@target_char_id) + ".";
        mes "Party: " + strcharinfo(1, .@target_char_id) + ".";
        mes "Guild: " + strcharinfo(2, .@target_char_id) + ".";
    }

    close;
}

      In this script:

      • This script defines an NPC named InfoNPC that, when clicked, provides information about the player interacting with it and optionally about another player.
      • Initial Dialogue: The NPC first greets the player and displays their character name and current map using strcharinfo(0) and strcharinfo(3) respectively.
      • Party and Guild Info: It then checks if the player is in a party or guild using strcharinfo(1) and strcharinfo(2). If the returned string is not empty, it displays the party/guild name; otherwise, it states that the player is not in one.
      • Targeted Player Info: After a next; command, the NPC prompts the player to enter the name of another character. It uses getcharid(0, .@target_name$) to get the character ID of the entered name.
      • Displaying Target Info: If a valid character ID is found (meaning the player is online), it proceeds to display the target player's map, party, and guild information using strcharinfo with the char_id parameter. If the target is not found, an appropriate message is displayed.

      This demonstrates how strcharinfo can be used to retrieve various string-based details about characters, both the invoking one and others, making scripts more dynamic and interactive.
