      The `getlook` Function in rAthena Scripting

      The getlook function in rAthena scripting is used to retrieve information about a character's current visual appearance.

      ‚ú® What is getlook?

      The getlook function returns the numerical value for a specified visual aspect of the invoking character. If a char_id is provided, it returns the look value for that specific player.

      üéØ Purpose of getlook

      The primary purpose of getlook is to allow scripts to behave differently based on a character's appearance. For example, you can create an NPC that interacts uniquely with players wearing a specific hair color or headgear.

      üìù Syntax of getlook

      The syntax for getlook is as follows:

      getlook(<type>{,<char_id>})

      ‚öôÔ∏è Syntax Parameter Explanation

      • <type>: This parameter specifies which aspect of the character's appearance you want to retrieve. It can be a number or a constant defined in src/map/script_constants.hpp. Valid look types include:
        • LOOK_BASE - Base sprite
        • LOOK_HAIR - Hairstyle
        • LOOK_WEAPON - Weapon
        • LOOK_HEAD_BOTTOM - Lower Headgear
        • LOOK_HEAD_TOP - Upper Headgear
        • LOOK_HEAD_MID - Middle Headgear
        • LOOK_HAIR_COLOR - Hair color
        • LOOK_CLOTHES_COLOR - Clothes color
        • LOOK_SHIELD - Shield
        • LOOK_SHOES - Shoes
        • LOOK_BODY2 - Body style
      • {,<char_id>}: This is an optional parameter. If provided, the function will return the look value for the character identified by this ID. If omitted, it defaults to the invoking character.

      üí° How to use getlook

      You use getlook by calling it with the desired look type. The function will return an integer representing the current value for that look type. You can then use this value in conditional statements or store it in a variable.

      Example Code:
      //===== rAthena Script =======================================
//= AppearanceChecker
//===== By: rAthena AI Assistant ============================
//= Checks a player's hair color and responds accordingly.
//============================================================
prontera,156,180,4	script	AppearanceChecker#MAIN	912,{
    mes "Hello, adventurer! I can tell you something about your appearance.";
    next;

    .@hairColor = getlook(LOOK_HAIR_COLOR);

    if (.@hairColor == 0) {
        mes "Your hair color is the default. It suits you!";
    } else if (.@hairColor == 1) {
        mes "Ah, a lovely shade of red! Very striking.";
    } else if (.@hairColor == 2) {
        mes "Is that a vibrant green? How unique!";
    } else {
        mes "Your hair color is quite interesting, but I don't have a special comment for it.";
    }
    close;
}

      In this script:

      This sample script, named AppearanceChecker. When a player interacts with it:

      • The NPC greets the player.
      • It then uses getlook(LOOK_HAIR_COLOR) to retrieve the player's current hair color ID and stores it in the temporary NPC variable .@hairColor.
      • A series of if-else if statements check the value of .@hairColor:
        • If the hair color is 0 (typically black/default), a specific message is displayed.
        • If it's 1 (often red), another message appears.
        • If it's 2 (often green), a different message is shown.
        • For any other hair color, a general comment is made.
      • Finally, the dialogue closes.

      This demonstrates how getlook can be used to create dynamic NPC interactions based on a player's visual attributes.
