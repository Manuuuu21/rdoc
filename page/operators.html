      Operators

      Operators in rAthena scripting are used to perform operations on variables and numbers. They are categorized into several types, each serving a different purpose. Let's explore them:

      ‚ûï‚ûñ‚úñÔ∏è‚ûó Arithmetic Operators

      These operators perform basic mathematical calculations:

      • + (Addition): Adds two numbers. If used with strings, it concatenates them.
      • - (Subtraction): Subtracts two numbers.
      • * (Multiplication): Multiplies two numbers.
      • / (Division): Divides two numbers. Note that this is an integer division, meaning 7/2 equals 3.
      • % (Modulo): Returns the remainder of a division. For example, 7%2 equals 1.
      Example Code:
      // ----------------------------------------------------------
// Arithmetic Operators
// ----------------------------------------------------------
mes "
      ‚ûï‚ûñ‚úñÔ∏è‚ûó Arithmetic Operators:
      ";
set .@num1, 10;
set .@num2, 3;

mes "Addition (10 + 3): " + (.@num1 + .@num2);
mes "Subtraction (10 - 3): " + (.@num1 - .@num2);
mes "Multiplication (10 * 3): " + (.@num1 * .@num2);
mes "Division (10 / 3 - integer division): " + (.@num1 / .@num2);
mes "Modulo (10 % 3): " + (.@num1 % .@num2);

      ‚öñÔ∏è Conditional Operators

      These operators are used for comparisons and return 1 (True) if the condition is met, and 0 (False) otherwise. They work with numbers and strings (though string comparison is limited to equality checks).

      • == (Equal to): True if both sides are equal.
      • => (Greater than or equal to): True if the first value is greater than or equal to the second.
      • <= (Less than or equal to): True if the first value is less than or equal to the second.
      • > (Greater than): True if the first value is greater than the second.
      • < (Less than): True if the first value is less than the second.
      • != (Not equal to): True if the first value is not equal to the second.
      • && (Logical AND): True if both sides are true.
      • || (Logical OR): True if either side is true.
      Example Code:
      // ----------------------------------------------------------
// Conditional Operators
// ----------------------------------------------------------
mes "
      ‚öñÔ∏è Conditional Operators:
      ";
set .@valA, 5;
set .@valB, 10;
set .@strA$, "Hello";
set .@strB$, "World";

if (.@valA < .@valB) {
    mes .@valA + " is less than " + .@valB + ". (True)";
}

if (.@strA$ == "Hello") {
    mes "String A is 'Hello'. (True)";
}

if (.@valA > 0 && .@valB < 20) {
    mes "ValA is positive AND ValB is less than 20. (True)";
}

if (.@valA == 1 || .@valB == 10) {
    mes "ValA is 1 OR ValB is 10. (True)";
}

      üßÆ Logical Bitwise Operators

      These operators work only on numbers, manipulating their binary representations:

      • << (Left Shift): Shifts bits to the left, equivalent to multiplying by powers of 2.
      • >> (Right Shift): Shifts bits to the right, equivalent to dividing by powers of 2.
      • & (Bitwise AND): Sets bits that are active in both arguments. Useful for bit-masks.
      • | (Bitwise OR): Sets a binary position to 1 if the binary position of one of the numbers is 1. Useful for setting multiple options in one variable.
      • ^ (Bitwise XOR): Sets a binary position to 1 if numbers have different values in that position, otherwise 0. Useful for toggling bits.
      Example Code:
      // ----------------------------------------------------------
// Logical Bitwise Operators
// ----------------------------------------------------------
mes "
      üßÆ Logical Bitwise Operators:
      ";
set .@flags, 1; // 0001
set .@flags, .@flags | 4; // 0101 (set bit 2)
mes "Flags after setting bit 2 (value 4): " + .@flags;

if (.@flags & 1) {
    mes "Bit 0 (value 1) is set. (True)";
}

set .@shifted, 2 << 3; // 2 * 2^3 = 16
mes "Left shift (2 << 3): " + .@shifted;

set .@shifted, 16 >> 2; // 16 / 2^2 = 4
mes "Right shift (16 >> 2): " + .@shifted;

set .@flags, .@flags ^ 1; // Toggle bit 0 (1 -> 0)
mes "Flags after toggling bit 0 (value 1): " + .@flags;

      ‚ûñ‚ùï‚ùì Unary and Ternary Operators

      Unary operators in rAthena scripting perform operations on a single operand: the negation operator (-) reverses a number's sign, the logical NOT operator (!) inverts a boolean value (turning 0 into 1 and any non-zero value into 0), and the bitwise NOT operator (~) flips each bit in a number's binary representation. In contrast, the ternary operator (?:) is a conditional operator that takes three expressions, acting as a shorthand for simple if-else statements by evaluating a condition and returning one of two specified values based on whether the condition is true or false.

      • - (Negation): Reverses the sign of a number.
      • ! (Logical NOT): Reverses the boolean result of an expression (True becomes False, False becomes True).
      • ~ (Bitwise NOT): Reverses each bit in a number (one's complement).
      • ?: (Conditional Operator): A ternary operator that acts as a shorthand for simple if-else statements.
      Example Code:
      // ----------------------------------------------------------
// Unary and Ternary Operators
// ----------------------------------------------------------
mes "
      ‚ûñ‚ùï‚ùì Unary and Ternary Operators:
      ";
set .@value, 5;
mes "Negation (-5): " + (-.@value);

set .@condition, 0;
if (!.@condition) {
    mes "Logical NOT (!0 is True).";
}

set .@bitmask, 10; // Binary 1010
mes "Bitwise NOT (~10): " + (~.@bitmask);

set .@isMale, 1; // Assume 1 for male, 0 for female
mes "Gender: " + (.@isMale ? "Male" : "Female");
next;

mes "That concludes the operator demonstration!";
