      getarg

      The getarg function in rAthena scripting is used to retrieve arguments that are passed to a function or subroutine when it is called using the callfunc or callsub commands. It allows you to make your functions or subroutines dynamic, performing different actions based on the values provided during the call.

      How to use getarg? üìù

      You use getarg within the body of a function or subroutine to access the values that were supplied as arguments. The arguments are indexed starting from 0, meaning the first argument passed is getarg(0), the second is getarg(1), and so on.

      Purpose of getarg? üéØ

      The primary purpose of getarg is to enable reusability and flexibility in your scripts. Instead of writing separate blocks of code for similar tasks that only differ by a few values, you can write a single function/subroutine and pass those differing values as arguments. This makes your scripts more organized, efficient, and easier to maintain.

      Syntax of getarg? üìú

      The syntax for getarg is as follows:

      getarg(<index>{,<default_value>})

      Syntax Parameter Explain. üìñ

      <index>: This is a numerical value representing the position of the argument you want to retrieve. The indexing starts from 0 (the first argument).

      {,<default_value>}: This is an optional parameter. If the argument at the specified <index> was not provided when the function/subroutine was called, getarg will return this <default_value> instead of terminating the script with an error. If <default_value> is omitted and the argument is not found, the script will terminate.

      Example Code:
      //===== rAthena Script =======================================
//= Argument_Example
//===== By: rAthena AI Assistant ============================
//= Demonstrates the use of getarg with callfunc.
//============================================================
prontera,152,219,4	script	Argument_NPC	707,{
    mes "Hello! I can perform calculations for you.";
    next;

    // Example 1: Adding two numbers
    mes "Let's add two numbers.";
    input .@num1;
    input .@num2;
    .@result = callfunc("CalculateSum", .@num1, .@num2);
    mes "The sum of " + .@num1 + " and " + .@num2 + " is " + .@result + ".";
    next;

    // Example 2: Multiplying two numbers with a default value for the second argument
    mes "Now, let's multiply. If you only give one number, I'll multiply it by 5.";
    input .@factor1;
    .@product = callfunc("CalculateProduct", .@factor1);
    
    mes "The product is " + .@product + ".";
    close;
}

//===== rAthena Script =======================================
//= CalculateSum
//===== By: rAthena AI Assistant ============================
//= Function to calculate the sum of two arguments.
//============================================================
function	script	CalculateSum    {
    .@val1 = getarg(0);
    .@val2 = getarg(1);
    return .@val1 + .@val2;
}

//===== rAthena Script =======================================
//= CalculateProduct
//===== By: rAthena AI Assistant ============================
//= Function to calculate the product of two arguments, with a default.
//============================================================
function	script	CalculateProduct    {
    .@val1 = getarg(0);
    .@val2 = getarg(1, 5); // Default value of 5 if second argument is not provided
    return .@val1 * .@val2;
}

      In this script:

      The Argument_NPC script interacts with the player, allowing them to perform two types of calculations by calling separate functions: CalculateSum and CalculateProduct.

      • For the sum calculation, the script prompts the player for two numbers. These numbers are then passed as arguments to the CalculateSum function. Inside CalculateSum, getarg(0) and getarg(1) retrieve these values, add them, and return the sum.
      • For the multiplication, the script now directly prompts for a single number. This number is passed as the first argument to the CalculateProduct function. Since no second argument is explicitly provided in the callfunc command, the CalculateProduct function will utilize the default value of 5 (defined by getarg(1, 5)) for the second factor, multiplying the player's input by 5.
      • This demonstrates how getarg enables functions to be flexible, either accepting all provided arguments or falling back to default values when arguments are omitted, promoting code reusability.
