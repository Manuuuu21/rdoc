      input

      The input command in rAthena scripting is used to prompt a player to enter a value, which can be either a number or a string. This is particularly useful for creating interactive NPCs where player input is required.

      ü§î Purpose of input

      The primary purpose of the input command is to allow players to provide data to a script. This data can be a numerical value (e.g., for a guessing game, quantity input) or a text string (e.g., for entering a name, a password, or a specific phrase). It creates an input box on the client's screen, pausing the script until the player enters a value and confirms it.

      üìú Syntax of input

      The basic syntax for the input command is:

      input(<variable>{,<min>{,<max>}})

      ‚öôÔ∏è Syntax Parameter Explanation

      <variable>: This is the variable where the player's input will be stored.

      • If you use an integer variable (e.g., .@num), the input box will only allow numerical input.
      • If you use a string variable (e.g., .@str$), the input box will allow text input.

      {,<min>}: (Optional) This parameter sets the minimum allowed value for numerical input or the minimum length for string input. If the input is below this, a warning is displayed, and the script terminates.

      {,<max>}: (Optional) This parameter sets the maximum allowed value for numerical input or the maximum length for string input. If the input is above this, a warning is displayed, and the script terminates.

      By default, negative numbers are not allowed for numerical input to prevent exploits in poorly written scripts. The default min and max values can be configured in conf/script_athena.conf.

      Example Code:
      //===== rAthena Script =======================================
//= Input_Example
//===== By: rAthena AI Assistant ============================
//= Demonstrates the use of the 'input' command for both numerical
//= and string input, including min/max constraints.
//============================================================
prontera,152,219,4	script	Input NPC	670,{
    mes "Hello, adventurer! I'm here to test your input skills.";
    next;

    mes "First, let's try a number. Guess a number between 1 and 10.";
    next;

    input .@guess,1,10; // Player inputs a number between 1 and 10

    if (.@guess == 7) {
        mes "Well done! You guessed the correct number: " + .@guess + "!";
    } else {
        mes "Sorry, that's not the number I was thinking of. You entered: " + .@guess + ".";
    }
    next;

    mes "Now, let's try some text. Please type 'rAthena' (case-sensitive).";
    next;

    input .@text_input$,1,10; // Player inputs a string with length between 1 and 10

    if (.@text_input$ == "rAthena") {
        mes "Excellent! You typed it correctly: " + .@text_input$ + ".";
    } else {
        mes "Oops! You typed: '" + .@text_input$ + "'. Please try again next time.";
    }
    next;

    mes "That's all for today. Farewell!";
    close;
}

      In this script:

      This sample script demonstrates two main uses of the input command:

      • Numerical Input: The NPC first asks the player to guess a number between 1 and 10. The input .@guess,1,10; line ensures that only a number within this range can be entered. The script then checks if the guess is correct (7 in this case) and provides feedback.
      • String Input: Next, the NPC prompts the player to type the word 'rAthena'. The input .@text_input$,1,10; line restricts the input to a string between 1 and 10 characters long. The script then verifies if the entered string matches 'rAthena' exactly (case-sensitive) and responds accordingly.
