      NPC Label

      NPC Labels are used in scripting to define specific points in a script that can be jumped to or triggered under certain conditions. They are essential for controlling the flow of NPC (Non-Player Character) scripts, handling events, or organizing script logic.

      What is a Label in rAthena?

      A label is a named marker in a script that allows you to direct the script's execution to a specific point. Labels are commonly used for:

      • Event handling: Triggering code when specific events occur (e.g., OnNPCKillEvent, OnInit)
      • Script flow control: Jumping to specific sections of the script using commands like goto.
      • Event-driven NPC interactions: Executing code when a player interacts with an NPC, kills a monster, or triggers other in-game events.

      Labels are defined by a name followed by a colon (:) and are case-insensitive in rAthena (e.g., OnInit and oninit are treated the same).

      üìù Syntax. A label is written as:
      LabelName:
      // Code here
      end;

      Example Code:
      // OnInit: Triggered when scripts are loaded (server start, @reloadscript)
OnInit:
    announce "ServerEvents NPC: Scripts have been loaded!",bc_blue;
    // You can also initialize timers here, e.g., initnpctimer;
end;

      üöÄ Script Commands That Utilize Labels

      Labels are essential for several control flow commands:

      • goto <label>;: This command forces the script to immediately jump to the specified label, continuing execution from that point.
      • menu "<option_text>",<target_label>{,...};: This command presents a selectable menu to the player. When an option is chosen, the script jumps to its corresponding target_label.
      • doevent "<NPC object name>::<event label>";: This command starts a new execution thread in a specified NPC object at the given label. The original script continues to run concurrently.
      • donpcevent "<NPC object name>::<event label>";: Similar to doevent, but it invokes the event code within another NPC or NPCs. If the NPC name is omitted (::OnLabel), it invokes the event for all NPCs with that label.
      • bindatcmd "<command>","<NPC object name>::<event label>";: This command links a custom atcommand to an NPC event label, causing the event to trigger when the atcommand is used.

      ‚ú® Special Labels

      rAthena also defines several special labels that are automatically triggered by specific server events or conditions:

      • OnClock<hour><minute>: / OnMinute<minute>: / OnHour<hour>: / On<weekday><hour><minute>: / OnDay<month><day>:: These labels execute when the server clock reaches a specified time or date.
      • OnInit:: Executes every time scripts are loaded, including server startup or when @reloadscript is used.
      • OnInterIfInit:: Executes when the map server connects to a char server.
      • OnInterIfInitOnce:: Executes only once when the map server initially connects to a char server.
      • OnAgitStart: / OnAgitEnd: / OnAgitInit: (and their 2 and 3 variants for WoE SE/TE): Triggered by War of Emperium events.
      • OnInstanceInit:: Executes when an instance is created via instance_create.
      • OnInstanceDestroy:: Executes when an instance is destroyed.
      • OnTouch:: Triggers when a player walks into an NPC's defined trigger area (if no OnTouch: label is present, execution starts from the beginning of the NPC code).
      • OnTouch_:: Similar to OnTouch, but only runs one instance at a time.
      • OnTouchNPC:: Triggers when a monster walks into an NPC's defined trigger area.
      • OnPCLoginEvent: / OnPCLogoutEvent: / OnPCBaseLvUpEvent: / OnPCJobLvUpEvent: / OnPCDieEvent: / OnPCKillEvent: / OnNPCKillEvent: / OnPCLoadMapEvent:: Triggered by specific player-related events.
      • OnWhisperGlobal:: Triggers when a player whispers the NPC.
      • OnNaviGenerate:: Triggers when running the map-server-generator binary.
      • On<label name>:: General special labels used with Mob scripts or commands that link to a mob/NPC event.
      Example Code:
      //===== rAthena Script =======================================
//= Special Labels Example
//===== By: rAthena AI Assistant ============================
//= Demonstrates various special labels and their triggers.
//============================================================

// Floating NPC for server-wide or general events
-	script	ServerEvents	-1,{

    // OnInit: Triggered when scripts are loaded (server start, @reloadscript)
OnInit:
    announce "ServerEvents NPC: Scripts have been loaded!",bc_blue;
    // You can also initialize timers here, e.g., initnpctimer;
    end;

    // OnClock: Triggered at a specific time (e.g., 00:00 server time)
OnClock0000:
    announce "ServerEvents NPC: It's midnight! A new day begins.",bc_yellow;
    end;

    // OnHour: Triggered at the start of a specific hour (e.g., 06:00 AM)
OnHour06:
    announce "ServerEvents NPC: Good morning! The sun is rising.",bc_green;
    end;

    // OnMinute: Triggered at a specific minute of every hour (e.g., at :30 past every hour)
OnMinute30:
    // This can be very spammy if not used carefully.
    // announce "ServerEvents NPC: It's :30 past the hour!",bc_blue;
    end;

    // OnAgitStart: Triggered when War of Emperium (FE) starts
OnAgitStart:
    announce "ServerEvents NPC: War of Emperium has begun! Prepare for battle!",bc_woe;
    end;

    // OnAgitEnd: Triggered when War of Emperium (FE) ends
OnAgitEnd:
    announce "ServerEvents NPC: War of Emperium has ended. Good fight, everyone!",bc_woe;
    end;

    // OnInterIfInit: Triggered when map server connects to char server
OnInterIfInit:
    debugmes "ServerEvents NPC: Map server connected to char server.";
    end;

    // OnInterIfInitOnce: Triggered only once when map server initially connects to char server
OnInterIfInitOnce:
    debugmes "ServerEvents NPC: Initial connection to char server established.";
    end;

    // OnPCLoginEvent: Triggered when a player logs in
OnPCLoginEvent:
    announce "ServerEvents NPC: Welcome, " + strcharinfo(0) + "!",bc_self;
    end;

    // OnPCLogoutEvent: Triggered when a player logs out
OnPCLogoutEvent:
    debugmes "ServerEvents NPC: " + strcharinfo(0) + " has logged out.";
    end;

    // OnPCBaseLvUpEvent: Triggered when a player's Base Level increases
OnPCBaseLvUpEvent:
    announce "ServerEvents NPC: Congratulations, " + strcharinfo(0) + ", on reaching Base Level " + BaseLevel + "!",bc_blue|bc_self;
    end;

    // OnPCJobLvUpEvent: Triggered when a player's Job Level increases
OnPCJobLvUpEvent:
    announce "ServerEvents NPC: Well done, " + strcharinfo(0) + ", on reaching Job Level " + JobLevel + "!",bc_blue|bc_self;
    end;

    // OnPCDieEvent: Triggered when a player dies
OnPCDieEvent:
    announce "ServerEvents NPC: " + strcharinfo(0) + " has fallen in battle!",bc_red|bc_self;
    end;

    // OnPCKillEvent: Triggered when a player kills another player
OnPCKillEvent:
    announce "ServerEvents NPC: " + strcharinfo(0) + " has defeated " + rid2name(killedrid) + "!",bc_red;
    end;

    // OnNPCKillEvent: Triggered when a player kills a monster without a specific event label
OnNPCKillEvent:
    announce "ServerEvents NPC: " + strcharinfo(0) + " killed a " + getmonsterinfo(killedrid,MOB_NAME) + ".",bc_blue;
    end;

    // OnWhisperGlobal: Triggered when a player whispers this NPC
OnWhisperGlobal:
    mes "ServerEvents NPC: You whispered: " + @whispervar0$;
    close;
    end;
}

// Standard NPC for direct player interaction and touch events
prontera,150,185,4	script	InteractionNPC	767,{
    mes "Hello! Click me or walk near me to see different interactions.";
    close;

    // OnTouch: Triggered when a player walks into the NPC's trigger area (if defined)
    // This NPC is defined with a default trigger area (4 cells in each direction around 150,185)
OnTouch:
    mes "InteractionNPC: You touched my trigger area!";
    close;
    end;

    // OnTouchNPC: Triggered when a monster walks into the NPC's trigger area
OnTouchNPC:
    // For this to work, the NPC needs a trigger area defined (e.g., prontera,150,185,4 script InteractionNPC 767,5,5,{...})
    // The 'getattachedrid()' in this context would return the GID of the monster.
    // announce "InteractionNPC: A monster touched me! GID: " + getattachedrid(),bc_blue;
    end;
}

      In this script:

      This rAthena script serves as a comprehensive demonstration of various special labels and NPC interaction types, featuring a floating ServerEvents NPC that manages server-wide and player-specific events such as script loading, time-based triggers, War of Emperium phases, and player login/logout/level-up/kill events, while also including a standard InteractionNPC positioned in Prontera that handles direct player clicks and area-based triggers for both players and monsters, showcasing different ways NPCs can react to in-game occurrences without direct interaction.
