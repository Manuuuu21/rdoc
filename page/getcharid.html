      getcharid()

      What is getcharid? üÜî

      The getcharid function returns a unique ID number associated with a character, their party, guild, or account. It can retrieve this ID for the invoking character or for a specified player by name.

      How to use getcharid? üí°

      You use getcharid by specifying the type of ID you need (e.g., character ID, account ID, party ID) and optionally providing a character's name if you're looking up information for someone other than the invoking player. It's crucial for identifying players and their associated groups within scripts.

      Purpose of getcharid? üéØ

      The primary purpose of getcharid is to obtain unique identifiers for characters and their related entities (parties, guilds, accounts). These IDs are essential for many script commands and functions that require a specific target, allowing scripts to interact with players, manage groups, or store player-specific data accurately. It also helps determine if a script is being run by an actual player.

      Syntax of getcharid? üìù

      • getcharid(<type>{"<character name>"})
      • getcharid(<type>{,<char_id>})

      Syntax Parameter Explain üß©

      • <type>: This parameter specifies the kind of ID number you want to retrieve. It can be one of the following integer values:
        • 0: Character ID (unique ID for the character itself).
        • 1: Party ID (unique ID for the party the character is in).
        • 2: Guild ID (unique ID for the guild the character is in).
        • 3: Account ID (unique ID for the account the character belongs to).
        • 4: Battle Ground ID (unique ID for the battleground the character is in).
        • 5: Clan ID (unique ID for the clan the character is in).
      • "<character name>" (Optional): If provided, the function will attempt to retrieve the ID for the character with this specific name. If omitted, it defaults to the invoking character.
      • <char_id> (Optional): Alternatively, you can provide a character's ID to retrieve information about that specific player.

      If the requested information (e.g., party ID for a character not in a party, or a character name not found) is not available, the function will return 0.

      Example Code:
      //===== rAthena Script =======================================
//= GetCharID Sample
//===== By: rAthena AI Assistant ============================ 
//= Demonstrates how to use the getcharid function to retrieve
//= various player and group IDs.
//============================================================
prontera,156,180,4	script	ID_Checker	910,{
    mes "Hello! I can tell you about your IDs.";
    next;

    // Get and display the invoking character's IDs
    .@char_id = getcharid(0);
    .@account_id = getcharid(3);
    .@party_id = getcharid(1);
    .@guild_id = getcharid(2);

    mes "Your Character ID is: " + .@char_id;
    mes "Your Account ID is: " + .@account_id;

    if (.@party_id != 0) {
        mes "Your Party ID is: " + .@party_id + " (Name: " + getpartyname(.@party_id) + ")";
    } else {
        mes "You are not currently in a party.";
    }

    if (.@guild_id != 0) {
        mes "Your Guild ID is: " + .@guild_id + " (Name: " + getguildname(.@guild_id) + ")";
    } else {
        mes "You are not currently in a guild.";
    }
    next;

    // Demonstrate checking for a specific player's ID (e.g., 'rAthena')
    mes "Let's check the Account ID for 'rAthena' (if online).";
    .@target_account_id = getcharid(3, "rAthena");

    if (.@target_account_id != 0) {
        mes "The Account ID for 'rAthena' is: " + .@target_account_id;
    } else {
        mes "'rAthena' is not found or not online.";
    }
    close;
}

      In this script:

      • Initialization: The NPC starts a dialogue and prepares to display various IDs.
      • Retrieving Invoking Character's IDs: It uses getcharid(0) to get the unique Character ID of the player who clicked the NPC. It uses getcharid(3) to get the Account ID of the player's account. It uses getcharid(1) to get the Party ID. If the player is not in a party, this will return 0. It uses getcharid(2) to get the Guild ID. If the player is not in a guild, this will return 0.
      • Displaying Information: The script then displays these retrieved IDs. For party and guild IDs, it includes a conditional check (if (.@party_id != 0)) to display the party/guild name using getpartyname or getguildname only if the player is actually part of one.
      • Checking a Specific Player: Finally, it demonstrates how to use getcharid(3, "rAthena") to attempt to retrieve the Account ID of a player named "rAthena". It then informs the user whether the player was found or not.

      This script effectively shows how getcharid can be used to identify players and their group affiliations, and how to handle cases where a player might not be part of a group or online.
