      callsub

      The callsub command in rAthena scripting is used to execute a subroutine within the current script. It allows you to reuse blocks of code multiple times without duplicating them, making your scripts more organized and efficient.

      Purpose of callsub üéØ

      The primary purpose of callsub is to enable code reusability within a single script. When you have a specific set of actions that need to be performed multiple times at different points in your script, you can define these actions under a label and then callsub to that label. This helps in:

      • Reducing script size and redundancy.
      • Improving readability and maintainability.
      • Allowing for modular script design within a single NPC.

      Unlike callfunc, which calls a separate NPC function, callsub keeps the execution flow within the same NPC script instance.

      Syntax of callsub üìù

      The basic syntax for the callsub command is:

      callsub <label>{,<argument>,...<argument>};

      Or, in its function form:

      callsub(<label>{,<argument>,...<argument>})

      Syntax Parameter Explanation üìñ

      <label>: This is the name of the label within the current script where the subroutine begins. Do not use quotes around the label name.

      {,<argument>,...<argument>}: These are optional parameters that you can pass to the subroutine. These arguments can be numbers, strings, or variables. Inside the subroutine, these arguments can be retrieved using the getarg() function, where getarg(0) refers to the first argument, getarg(1) to the second, and so on.

      After the subroutine finishes its execution, it should use the return command to transfer control back to the point where callsub was originally invoked.

      Example Code:
      //===== rAthena Script =======================================
//= ZenyChecker NPC
//===== By: rAthena AI Assistant ============================
//= Demonstrates the use of callsub for Zeny management.
//============================================================
prontera,152,219,4	script	ZenyChecker	701,{
    mes "Hello! I can help you with some transactions.";
    next;

    // Transaction 1: Buy a Red Potion for 50 Zeny
    callsub S_ProcessTransaction, 50, 501, 1; // Cost: 50 Zeny, Item ID: 501 (Red Potion), Amount: 1

    if (@transaction_success) {
        mes "You successfully bought a Red Potion!";
    } else {
        mes "You don't have enough Zeny for a Red Potion.";
    }
    next;

    // Transaction 2: Buy a Blue Potion for 100 Zeny
    callsub S_ProcessTransaction, 100, 505, 1; // Cost: 100 Zeny, Item ID: 505 (Blue Potion), Amount: 1

    if (@transaction_success) {
        mes "You successfully bought a Blue Potion!";
    } else {
        mes "You don't have enough Zeny for a Blue Potion.";
    }
    close;


S_ProcessTransaction:
    // Arguments:
    // getarg(0): Zeny cost
    // getarg(1): Item ID
    // getarg(2): Item amount

    .@cost = getarg(0);
    .@item_id = getarg(1);
    .@item_amount = getarg(2);

    @transaction_success = 0; // Initialize success flag

    if (Zeny >= .@cost) {
        Zeny -= .@cost;
        getitem .@item_id, .@item_amount;
        @transaction_success = 1;
    }
    return;
}

      In this script:

      The NPC ZenyChecker has a main script that initiates two separate transactions.

      • Each transaction calls the S_ProcessTransaction subroutine using callsub.
      • The S_ProcessTransaction subroutine takes three arguments: the Zeny cost, the item ID, and the item amount.
      • Inside S_ProcessTransaction, it checks if the player has enough Zeny. If so, it deducts the Zeny and gives the item, setting the @transaction_success temporary variable to 1. Otherwise, @transaction_success remains 0.
      • After the subroutine completes, control returns to the main script, which then checks the @transaction_success flag to determine the appropriate message to display to the player.
      • This demonstrates how callsub allows the Zeny and item handling logic to be centralized in one subroutine, avoiding repetition.
