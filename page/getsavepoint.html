      getsavepoint()

      üìú What is getsavepoint?

      The getsavepoint is a function that retrieves specific information about the invoking character's currently set save point. This is the location where a character respawns after death or when using a Butterfly Wing.

      üõ†Ô∏è How to Use and Purpose

      The primary purpose of getsavepoint is to access the map name, X-coordinate, and Y-coordinate of a player's save point. This is particularly useful in scripts where you might need to check a player's home location or create custom teleportation services that can send them back to their save point without using the standard warp "SavePoint",0,0; command.

      ‚å®Ô∏è Syntax

      The syntax for the command is as follows:

      getsavepoint(<information type>{,<char_id>})

      üìã Parameter Explanation

      • <information type>: This is a required numerical value that determines what piece of information you want to retrieve.
        • 0: Returns the map name as a string (e.g., "prontera").
        • 1: Returns the X-coordinate as a number.
        • 2: Returns the Y-coordinate as a number.
      • {,<char_id>}: This is an optional parameter. If you provide a character's ID, the function will retrieve the save point information for that specific character instead of the one interacting with the NPC.
      Example Code:
      //===== rAthena Script =======================================
//= Save Point Checker
//===== By: rAthena AI Assistant ============================
//= This NPC will check and inform the player of their
//= current save point location.
//============================================================
prontera,155,185,5	script	Save Point Checker	914,{
    mes "[Save Point Checker]";
    mes "Hello there, adventurer!";
    mes "Would you like me to check where your current save point is?";
    next;

    if (select("Yes, please!":"No, thank you.") == 2) {
        mes "[Save Point Checker]";
        mes "Alright, come back if you change your mind.";
        close;
    }

    // Retrieve the save point information into variables
    .@map$ = getsavepoint(0);
    .@mapX = getsavepoint(1);
    .@mapY = getsavepoint(2);

    mes "[Save Point Checker]";
    mes "Let me see...";
    mes "Your save point is set at:^0000FF " + .@map$ + " (" + .@mapX + "," + .@mapY + ")^000000.";
    close;
}

      In this script:

      • The NPC greets the player and asks if they want to know their save point location.
      • A select menu provides the player with a "Yes" or "No" option. If the player chooses "No," the script ends.
      • If the player chooses "Yes," the script proceeds to call the getsavepoint function three times:
        • getsavepoint(0) retrieves the map name and stores it in the temporary string variable .@map$.
        • getsavepoint(1) retrieves the X-coordinate and stores it in the temporary integer variable .@mapX.
        • getsavepoint(2) retrieves the Y-coordinate and stores it in the temporary integer variable .@mapY.
      • Finally, the NPC uses the mes command to display the stored information to the player in a formatted message, showing the map name and coordinates in blue text for emphasis.
