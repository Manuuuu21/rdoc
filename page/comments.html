      Comments

      In rAthena scripting, you can use two types of comments to add notes or temporarily disable parts of your code.

      üìù Single-Line Comments

      Single-line comments are initiated with two forward slashes (//). Everything from these slashes to the end of the line is treated as a comment and is completely ignored by the script parser. This makes them ideal for adding brief notes or temporarily disabling a single line of code.

      Example Code:
      //===== rAthena Script =======================================
//= SingleLineCommentExample
//===== By: rAthena AI Assistant ============================
//= Demonstrates the use of single-line comments.
//============================================================

// This is another single comment.
// You can use the single comment to describe your code.
// So that, you can easily know the code how it works.

      In the example above, the text following // on each line within the script block is a comment. These comments are not executed by the server and serve only as explanatory notes for developers.

      Block comments start with /* and end with */. Any text placed between these two symbols, even across multiple lines, will be ignored by the script parser. This is useful for longer explanations or for commenting out larger sections of code.

      Example Code:
      /* This text,
 * no matter which new line you start
 * is ignored, until the following
 * symbol is encountered: 
 */

/*  
This text,
no matter which new line you start
is ignored, until the following
symbol is encountered.
*/

      In the example above, the text enclosed within /* and */ is a block comment. This entire section is skipped by the server when the script is executed, serving only as internal documentation.
