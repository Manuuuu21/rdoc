      navigateto

      The navigateto command in rAthena scripting is used to generate a navigation path for a character on the client-side. It helps players find their way to a specified map location or even a monster.

      üß≠ Purpose of navigateto

      The primary purpose of navigateto is to provide in-game navigation assistance to players. It can guide them to quest objectives, specific NPCs, or even monster spawns, enhancing the player experience by reducing the time spent searching for locations.

      üìù Syntax of navigateto

      The syntax for the navigateto command is as follows:

      navigateto("<map>"{,<x>,<y>,<flag>,<hide_window>,<monster_id>,<char_id>});

      üîç Syntax Parameter Explanation

      • <map>: The name of the destination map. This is a string.
      • <x>: The X-coordinate on the destination map.
      • <y>: The Y-coordinate on the destination map.
      • <flag>: Specifies how the client will calculate the route. Valid flags are:
        • NAV_NONE: No services.
        • NAV_AIRSHIP_ONLY: Airship only.
        • NAV_SCROLL_ONLY: Scroll only.
        • NAV_AIRSHIP_AND_SCROLL: Airship and Scroll.
        • NAV_KAFRA_ONLY: Kafra only.
        • NAV_KAFRA_AND_AIRSHIP: Kafra and Airship (default if omitted).
        • NAV_KAFRA_AND_SCROLL: Kafra and Scroll.
        • NAV_ALL: All services.
      • <hide_window>: Specifies whether to display (0) or hide (1) the navigation window. By default, the window is hidden (1).
      • <monster_id>: The ID of a monster to navigate to. This is used in combination with a map name to tell the navigation system that the player has reached the desired mob.
      • <char_id>: The character ID of the player for whom the navigation will be generated. If omitted, it defaults to the attached character.
      Example Code:
      //===== rAthena Script =======================================
//= Navigation Guide
//===== By: rAthena AI Assistant ============================
//= Guides the player to a specific location or monster.
//============================================================
prontera,156,180,4	script	Navigation Guide	67,{
    mes "Hello, adventurer! Where would you like to go?";
    next;

    menu "Go to Izlude (120,120)",L_Izlude,"Find a Poring",L_Poring,"Cancel",L_Cancel;

L_Izlude:
    mes "Alright, I will show you the way to Izlude.";
    close2;
    // Navigate to Izlude map at coordinates 120,120, using Kafra and Airship services, and hide the navigation window.
    navigateto("izlude",120,120,NAV_KAFRA_AND_AIRSHIP,1);
    end;

L_Poring:
    mes "Let's find a Poring for you!";
    close2;
    // Navigate to a Poring (monster ID 1002) on the Prontera map at random coordinates, showing the navigation window.
    navigateto("prt_fild08",0,0,NAV_ALL,0,1002);
    end;

L_Cancel:
    mes "Understood. Come back anytime!";
    close;
}

      In this script:

      • The NPC offers the player two navigation options: going to a specific location (Izlude) or finding a monster (Poring).
      • If the player chooses "Go to Izlude", the navigateto command is used with the map name "izlude", coordinates 120,120, NAV_KAFRA_AND_AIRSHIP flag (allowing navigation via Kafra and Airship), and 1 to hide the navigation window after activation.
      • If the player chooses "Find a Poring", the navigateto command is used with the map name "prt_fild08", coordinates 0,0 (for a random spawn location), NAV_ALL flag (allowing all services), 0 to show the navigation window, and 1002 as the monster_id for Poring.
      • The close2 command is used before navigateto to close the NPC dialogue window while allowing the script to continue and the navigation to activate.
